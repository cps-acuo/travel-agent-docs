## Purpose 
The Travel section encompasses all daily operations for managing packages and bookings.
</Step>
<Step>
## Key Components

The Travel section includes several key components:

- **Travel Packages**: Create and manage packages that can be booked
- **Travel Bookings**: Track individual bookings, a booking consists of 
  - **Traveler**: The individuals that are included in the booking along with the prices and activities 
  - **Traveler addons**: An addon that traveler might have added to their bookings
  - **Payments**: Payment schedule for each booking
- **Travelers**: Manage information on individual travelers across packages and bookings
- **Journal**: Create travel journals with travelers that have been sold a package. Once the journal is posted, bookings are created.
- **Reports**: Generate reports for flights, hotels, passports, and more
- **Payments**: Handle all payment and collection of funds for bookings
</Step>
<Step>
## Getting Started

To get started with the Travel section:

1. Setup - before starting to create Travel Packages, you need to understand some basic setup. This includes configuring 
- *Parameter*
- *Action*
- *Payment Mehtod*
- *Destination*
- *Brand*
- *Item*
the necessary environment variables and ensuring that the backend is running.
2. [Travel Packages](./travel-packages) to understand how to create and manage travel offerings
3. [Travel Bookings](./travel-bookings) for your customers
4. Familiarize yourself with the [Travelers](./travelers) management system
5. Understand how the [Journal](./journal) is used to add travelers to a package and create bookings.

Each section provides detailed instructions for efficient travel management operations.
</Step>
