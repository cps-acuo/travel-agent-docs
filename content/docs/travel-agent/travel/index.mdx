## Purpose 
The Travel section is where you manage your day-to-day work — creating tour or event packages, handling bookings, and processing payments.
</Step>
<Step>
## Key Components

Travel section includes several key components:

- **Travel Packages**: Create and manage packages that can be booked including, Prices, Payments Options, Actions and availability for each offer witin the action.
- **Travel Bookings**: Track individual bookings, a booking consists of 
  - **Traveler/Attendee**: The individuals that are included in the booking along with the prices and activities 
  - **Traveler/Attendee activity**: An activity that the traveler or attendee has purchased with bookings along with prices and dates
  - **Payments**: Payment schedule for each booking
- **Travelers/Attendees**: Manage information on individuals across packages and bookings
- **Journal**: Create travel journals with travelers that have been sold a package. Once the journal is posted, bookings are created.
- **Reports**: Generate reports for flights, hotels, passports, and more
- **Payments**: Handle all payment and collection of funds for bookings
</Step>
<Step>
## Getting Started

To get started with the Travel section:

1. **Setup** - before starting to create Travel Packages, you need to understand some basic setup. This includes configuring the following:
   - *Parameter*
   - *Action*
   - *Payment Method*
   - *Destination*
   - *Brand*
   - *Item*
2. Review [Travel Packages](./travel-packages) to understand how to create and manage travel offerings
3. Review [Travel Bookings](./travel-bookings) for your customers
4. Familiarize yourself with the [Travelers](./travelers) management system
5. Understand how the [Journal](./journal) is used to add travelers to a package and create bookings.

Each section provides detailed instructions for efficient travel management operations.
</Step>
</Steps>
