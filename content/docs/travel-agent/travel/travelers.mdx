<Step>
## Booking Traveler/Attendee
The recommended way to book a new traveler or attendee is through the **[Journal](./journal)** page. The journal serves as the primary entry point for registering individuals and creating bookings in a single workflow.

Alternatively, you can book a traveler/attendee directly from the Bookings page by clicking the **New Traveler** button in the associated lookup grid while editing a booking. This approach automatically links the new record to the selected booking.


</Step>

<Step>
## Editing a Traveler/Attendee and Viewing Related Information
To edit an existing traveler/attendee, simply click on their name in the list. This will take you to the traveler/attendee's detail page, where you can update their information and view related data across several tabs.

![The Traveler/Attendee edit form](/images/travel-agent/travel-travelers-edit-john-smith.png)

The traveler/attendee detail view is organized into sections:
- **General**: Contains the main contact and personal information for the traveler/attendee.
- **Traveler Actions**: Lists any specific actions or activities associated with the traveler/attendee.

The user can also see related information in the traveler/attendee detail view, where the options include:
- **Audit History**: Shows a history of changes made to the traveler/attendee's information, if that feature is enabled.
- **Bookings**: Shows booking related to the traveler/attendee where one can see all information about the booking.
- **Journal Line**: Journal line related to the traveler/attendee that was used for posting the booking.



### Traveler/Attendee Information
When creating a new traveler/attendee, you'll need to provide information. 

Key fields include:
- **First Name & Last Name**: The traveler/attendee's full name.
- **Contact Information**: Email and Phone Number.
- **SSN**: The traveler/attendee's Social Security Number, if needed.
- **Address**: The traveler/attendee's address.
- **Passport Information**: The traveler/attendee's passport details if needed.
</Step>
